# Data Dictionary

## Advisory evaluation evidence

- `verification/safetydb/summary.json`: row-aggregated SafetyDB action-separation results, including claim inventory, edge dispositions, suite objectives, comparator outcomes, resilience, evolution, runtimes, and frozen input digests. The source-level SafetyDB inputs are not redistributed; this file supports a digest-bound historical case study rather than a full source-level reproduction claim.
- `verification/safetydb/README.md`: directory-local scope statement for the historical SafetyDB evidence and the optional authorized rerun path.
- `verification/safetydb_external_summary.json`: independently validated inventory and solver summary for the same historical PyPI benchmark.
- `scripts/run_observational_actions.py`: optional fail-closed source-level rerun entry point for evaluators who separately provide the non-redistributed claim, edge, and release-universe JSONL inputs plus matching analysis-gate attestations.
- `benchmarks/synthetic_advisory/CONSTRUCTION.md`: deterministic construction record for the synthetic controls, including design dimensions, scale boundary, and overfitting boundary.
- `benchmarks/synthetic_advisory/claims.jsonl`: redistributed synthetic advisory claims used for source-level semantic controls.
- `benchmarks/synthetic_advisory/edges.jsonl`: redistributed synthetic claim-pair controls with expected `EQUIVALENT`, `DIVERGENT`, or `ABSTAIN` outcomes.
- `benchmarks/synthetic_advisory/release_universes.jsonl`: redistributed package release histories for the synthetic controls.
- `verification/synthetic_advisory_benchmark.json`: replayed synthetic fixture hashes, action-trace digest, comparator decision/abstention rates, negative-control false-divergence count, divergent-control misses, and suite certificates.
- `benchmarks/public_advisory_overlap/`: redistributed source-level GHAD/PyPA overlap fixture with six public advisory records, three paired edges, and two frozen PyPI release universes. It is a construct witness that affected-set equality can hide target/action differences, not a prevalence sample.
- `verification/public_advisory_overlap.json`: replayed public-overlap fixture hashes, source-record parse check, affected-set decisions, action decisions, witness counts, and edge-result digest.
- `source_manifest.csv`: fixed public advisory repositories and commits supported by the source adapters.
- `external_resources.csv`: external corpora, licenses, immutable locators, and inclusion policy.

## Optimization benchmarks

- `benchmarks/orlib50/inputs/*.txt`: the 50 complete OR-Library SCP instances from sets 4--6 and A--E.
- `benchmarks/orlib50/orlib_results.csv`: one result row per instance for greedy, LP-guided rounding, and bounded HiGHS.
- `benchmarks/orlib50/orlib_report.json`: full row-level metrics, known optima, input SHA-256 values, summary statistics, and canonical report digest.
- `verification/method_validation.json`: compact controlled smoke-campaign evidence refreshed by the reproduction command.
- `verification/method_validation_release.json`: controlled exhaustive, MILP, interval, weighted-interval, and cross-certification campaign summary.
- `verification/mutation*.json`: certificate corruption campaigns and rejection totals.
- `verification/evolutionary.json`: cost perturbation, witness-failure, and incremental-update experiments.
- `verification/interval_scalability.json`: broad interval scalability measurements through 100,000 releases and obligations.
- `verification/ordered_interval_scalability.json`: ordered right-endpoint scalability measurements through 100,000 releases and obligations.
- `verification/adversarial_scalability.json`: dense-overlap interval scalability control over 20,000 obligations and 10,000 releases.

## Reproduction evidence

- `verification/method_smoke.json`: seeded short cross-oracle replay generated by the full reproduction command.
- `verification/result_validation.json`: independent consistency check over the bundled empirical reports.
- `verification/coverage.json` and `coverage.txt`: branch-aware execution coverage for tests and validators.
- `verification/test_attestation.json`: exact test command and outcome.
- `verification/audit_attestation.json`: anonymous release audit over the non-verification subject tree.
- `verification/release_attestation.json`: archive names, byte sizes, SHA-256 values, and release-gate statuses.
- Evaluator-local reproduction reports: end-to-end tests, validation, audit, deterministic packaging, and clean-extraction status emitted to the chosen output directory outside the project tree; excluded from deterministic release archives to avoid self-referential packaging.

## Traceability

- `reference_ledger.csv`: one row for each cited BibTeX key, its primary locator, use, and verification status.
- `config/baseline_lock.json`: frozen project-defined canonical action relation and comparator definitions, decision domain, metrics, denominators, and negative controls.
- `config/resource_lock.json`: public source commits, projection rules, registry families, and terminal-status policy.
`UNKNOWN`, failed registry acquisition, empty release histories, and unsupported syntax are typed outcomes. They are retained in evidence accounting and never serve as concrete witnesses.
